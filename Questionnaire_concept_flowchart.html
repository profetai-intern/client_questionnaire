AI 問卷建議邏輯流程圖 (完整版)

解釋問卷如何根據客戶的選擇，自動產生對應的解決方案建議。

起點：客戶提交問卷

背景盤點：了解業務目標

分析第一部分，為後續的建議提供重要的情境脈絡。

判斷基準：

這一步驟沒有自動判斷，主要是資訊收集，以幫助我們理解客戶的「為什麼」。

  • 第一題 (業務目標)：初步對應客戶需求。例如「預測趨勢」傾向 AutoML，「提取洞察」傾向 LLM。
  • 第二、三題 (開放式問題)：提供質化資訊，讓我們了解客戶期望的商業價值與具體痛點。

第一關：檢查數據類型 (最高優先級)

分析第四題「數據類型」的選項，這是最關鍵的指標。

判斷基準：

系統會檢查客戶是否勾選了「非結構化數據」。

因為處理文字、郵件、合約等非結構化資料是 LLM 的核心強項，所以這個選項具有一票否決權。

IF 成立 (已勾選)

➡️ 直接跳到最終結果，建議 LLM 應用。

IF 不成立

➡️ 進入第二關的判斷。

第二關：比較「情境分析」勾選數

計算第三部分「情境 A (預測)」 vs. 「情境 B (生成)」的勾選數量。

IF「情境 A」勾選數 >「情境 B」勾選數

判斷基準：客戶的需求更偏向數據預測與分類。

➡️ 建議 AutoML 平台。

IF「情境 B」勾選數 >「情境 A」勾選數

判斷基準：客戶的需求更偏向文字理解與內容生成。

➡️ 建議 LLM 語言模型應用。

IF 兩者勾選數相等 (且 > 0)

判斷基準：客戶的需求同時涵蓋了數據預測與文字處理。

➡️ 建議 AutoML + LLM 混合策略。

ELSE (未勾選任何情境)

判斷基準：客戶未提供足夠資訊進行初步判斷。

➡️ 顯示「尚無明確建議」。

終點：顯示客製化建議與總結
